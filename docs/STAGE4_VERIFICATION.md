# Stage 4 verification - the fix step (Layer 2)

Stage 4 is the first stage with **no dependency on a live model in its own logic**,
and it was therefore executed and verified in full here.

## A. Verified here (executed)
1. **Syntax** - `layer2.py` and `drilldown.py` compile.
2. **Rendering** - `render_layer2` produces the Layer 2 detail block from
   `SKILL.md`: severity/title header with id, rule id, and confidence; area;
   location; `What`; `Why`; the suggested fix; and the caveat.
3. **Trust framing** - `MECHANICAL` and `SUBSTANTIVE` labels are emitted correctly
   per `fix_kind`.
4. **Caveat** - the read-not-run note appears on every finding.
5. **Validation** - a malformed suggestion (empty / no before/after) is flagged
   with `[!]`; a well-formed one is not.
6. **Selection** - filtering by finding id and by severity returns the correct
   subset.
7. **JSON round-trip** - a `CombinedReport` dumped to JSON and reloaded renders
   identically. This is exactly the path `drilldown.py` uses (it reads a saved
   report), so the CLI is verified end to end apart from the file I/O.

All of the above passed.

## B. The boundary this stage does not cross
Stage 4 verifies the **format and framing** of a suggestion, not its
**correctness**. The suggestion text is generated by the checkers (Stages 2-3)
using a live model, and the tool never executes the code. So:
- a suggested fix is a reviewed proposal, not a verified result;
- substantive fixes are labelled for review; every fix carries the caveat;
- confirming a fix requires re-running the corrected script - which is the user's
  step, by design.

## C. What remains unverified (upstream)
Nothing in Stage 4 itself. Its **input** - the findings and their suggestion text -
comes from Stages 2-3, whose live behaviour is still unmeasured. In other words,
Stage 4 correctly presents whatever the checkers produce; whether the checkers
produce good findings and good suggestions is the Stage 2-3 measurement still to be
run against a live model.

## Conclusion
The drill-down layer is complete and fully verified: it renders selected findings
with correct trust framing, a validation flag, and the caveat, and it survives the
JSON round-trip the CLI relies on. It presents fixes; it does not certify them -
consistent with the tool's read-not-run design.
